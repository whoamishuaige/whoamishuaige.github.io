运维面试题小做(1)

Posted on 2019-02-19 | In devops , 运维面试题

  1. linux 如何挂载在window上的共享目录

    • 思考:本题考查linux基本命令操作方式，mount命令以及文件系统的管理，属于存储题。
    • 解答:mount -t cifs //192.168.1.3/server /mnt/server -o user=administrator,pass=123456
    • 详细解释: mount -t 选择filestystem的类型为cifs，原因是cifs是samber的默认文件系统格式。同时注意,linux必须安装samber服务,
      sudo apt-get install nfs-common
      sudo apt-get install smbfs       window上samber服务必须开启 关闭防火墙等操作 -o则是指定用户名和密码
      192.168.1.3是window的机器的ip地址server是共享文件夹名称。/mnt/server是指linux上创建的文件夹

  2. 查看http并发请求以及其tcp连接状态

    • 思考:本题考查netstat 命令用法，网络知识，管道awk命令用法，属于网络题
    • 解答：1. netstat -nt | awk ‘{++S[$NF]}END{for(a in S) print a,S[a]}‘
    • 详细解释：用awk的原因是,可以统计每一种不同的状态并且存储起来，netstat -tn 取出tcp状态并且不解析为机器名称 在统计完成以后，用for循环，将不同状态的的数量全部打印出来,在netstat -nt 以后，显示的内容默认为六列，其中最后一列为状态，状态有以下几种
    • 补充知识 还有ulimit -n 查看linux系统打开最大的文件描述符，这里默认1024，不修改这里web服务器修改再大也没用。若要用就修改很几个办法，这里说其中一个：
      修改/etc/security/limits.conf
      soft nofile 10240
      hard nofile 10240

  3. 查看当前系统每个ip的连接数

    • 思考：本体还是考查netstat用法，以及awk取值的方式,属于网络题
    • 回答：netstat -n |awk ‘/^tcp/ {print $5}’ |awk -F : ‘{print $1}’|sort |uniq -c |sort -rn
    • 详细解释: 必须用awk ‘/^tcp/‘这样的方式来取tcp否则最开始显示的几行都会被打印出来，后面的awk -f 是取ip而舍弃端口，sort |uniq -c |sort -rn 的方式则是在去重合并以及按照数字排列的常用方式

  4. shell下生成32为随机密码的方式

    • 思考： 考查命令的使用方式，考查linux系统伪设备，考查/dev/urandom,/dev/random的差别 属于存储题
    • 解答：cat /dev/urandom | head -1 | md5sum | head -c 32 >> /pass
    • 详细解释: 通过伪设备打印出来，并且经过一系列计算，并且取出前32位

  5. 统计出apache的access.log中访问量最多的5个IP

    • 思考： apache 服务器日志是可以自己定义的，所以肯定是默认的ip地址存放位置。本题考查的是对apache默认日志的熟悉程度,经过刚才的查找,发现apache的ip地址存放在每一条日志的首部。属于网络题
    • cat /var/log/access.log |awk ‘{print $1}’|sort |uniq -c |sort -rn |head -5
    • 详细解答：awk 取出ip 后面的sort |uniq|sort -rn 都是去重排序操作 head 则是取出前五内容

  6. rss和vsz的区别

    • 思考:这是一道典型的存储题。考查linux 内存管理方式以及一些术语的区别。
    • 解答:rss 实际使用内存 , vsz 虚拟内存 rss的计算方式
    • 详细解答。
      rss(实际使用内容) = 程序本身实际使用内容+堆栈交换分区内内存+共享库实际内存
vsz(虚拟内存) = 程序本身占有内存+堆栈分区占有内存+共享库占有内存
所以vsz>>>>rss

  8. 检测并修复/dev/hda5

    • 思考：本题考察文件存储方式，如何存储一个文件并且在出现问题之后尝试修复，属于存储题
    • 解答：fsck用来检查和维护不一致的文件系统。若系统掉电或磁盘发生问题，可利用fsck命令对文件系统进行检查,
    • 详细解答。fsck实际上是通盘扫面文件系统，尝试恢复数据

  9. linux 机器的开机启动流程

    • 思考:这道题目考察的点很多,但是总体来说,属于存储题目
    • 解答: 1:bios自检查 2:读取mbr 3：boot loader 4：使用grub配置 5:加载内核 6: init启动级别 7:启动内核 8:执行本地服务 9:执行/bin/login
    • 详细解答： 本体考查点有两个，一个是内核加载前做了什么，一个是内核加载以后做了什么。内核加载前操作是通用操作，只要使用特定芯片的机器，那么机器的一般设置都是这样，而linux机器启动以后，级别init启动级别，本地服务和/bin/login 执行注意前后就可以

  10. 符号链接和硬链接区别

    • 思考:本题属于存储题，考的是linux是如何存取文件
    • 解答: 软链接是创建的文件是指向文件本身，删除软连接对文件本身是没有影响的，修改软链接文件会修改文件本身，因为有指针指向。硬链接是相当于文件复制了一份
    • 详细解释: 在linux的存储设计中，文件名称只是代号，真正绝对文件存储的是inode，系统将文件名转换为inode，然后通过inode查看文件的具体数据,而设计链接的作用是解决文件共享使用,增加文件安全，节省存储等,当一个inode号对应多个文件名的时候,那么这个文件就是一个硬链接,所以硬连接的文件是独立的,删除其中一个不会影响其他,这设计的时候不能跨文件系统使用，而软链接的inode和源文件的inode是不同的，软链接的data block的内容指向原文件的inode
